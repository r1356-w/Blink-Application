# Blink — Flutter / GetX / FCM Professional Assessment

تطبيق Blink هو نسخة مصغرة واحترافية من تطبيق طلبات العملاء، تم بناؤه باستخدام تقنيات Flutter الحديثة مع التركيز على هندسة البرمجيات النظيفة (Clean Architecture) وتجربة المستخدم السلسة.

## التقنيات المستخدمة (Tech Stack)
- **Flutter**: أحدث إصدار مستقر.
- **GetX**: لإدارة الحالة (State Management)، التوجيه (Routing)، وحقن التبعيات (Dependency Injection).
- **Dio**: للتعامل مع طلبات الشبكة (Networking) مع دعم Interceptors.
- **Firebase Cloud Messaging (FCM)**: لاستقبال التنبيهات اللحظية.
- **Shared Preferences**: للتخزين المحلي المستمر.
- **Shimmer & CachedNetworkImage**: لتوفير تجربة تحميل بصرية ممتازة.

## هيكلية المشروع (Architecture)
يتبع المشروع نمط **Feature-First Architecture** لضمان سهولة الصيانة والتوسع:
- **Core**: يحتوي على الخدمات المشتركة، إعدادات الشبكة، السمة (Theme)، والترجمة.
- **Features**: مقسمة إلى ميزات مستقلة (Auth, Home, Store, Profile, Notifications)، وكل ميزة تحتوي على:
    - **Data**: النماذج (Models) ومصادر البيانات.
    - **Domain**: المستودعات (Repositories) ومنطق الأعمال.
    - **Presentation**: الواجهات (Screens)، العناصر (Widgets)، والمتحكمات (Controllers).
    - **Bindings**: لربط الميزة وحقن التبعيات الخاصة بها.

## الميزات الرئيسية
1.  **نظام المصادقة**: تسجيل دخول برقم الهاتف ورمز التحقق (OTP).
2.  **الصفحة الرئيسية**: عرض العروض (Banners) والمتاجر المميزة بشكل ديناميكي من الـ API.
3.  **تفاصيل المتجر**: عرض المنتجات مع دعم التحميل اللانهائي (Pagination).
4.  **الملف الشخصي**: إدارة بيانات المستخدم وتغيير لغة التطبيق (العربية/الإنجليزية).
5.  **التنبيهات**: عرض قائمة التنبيهات واستقبال تنبيهات FCM في المقدمة (Foreground).
6.  **دعم كامل للـ RTL**: واجهة مستخدم متوافقة تماماً مع اللغة العربية.

## خطوات التشغيل (Setup & Run)
1.  تحميل التبعيات:
    ```bash
    flutter pub get
    ```
2.  إعداد Firebase:
    - أضف ملف `google-services.json` في مسار `android/app/`.
    - أضف ملف `GoogleService-Info.plist` في مسار `ios/Runner/`.
3.  تشغيل التطبيق:
    ```bash
    flutter run
    ```

## ملاحظات حول التنبيهات (FCM)
- **في حالة الخلفية (Background)**: يتم التعامل مع التنبيهات عبر النظام وتظهر في شريط التنبيهات.
- **عند النقر**: تم تصميم النظام ليدعم الانتقال لصفحة معينة بناءً على بيانات التنبيه (Deep Linking).
- **الويب**: تم تحصين الكود ليعمل على المتصفح دون مشاكل برمجية رغم عدم دعم التنبيهات بشكل كامل في بيئة الاختبار.
